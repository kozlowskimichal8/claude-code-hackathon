⑩ Government customers silently receive Regular rates

Reflection

How We Used Claude Code

What Worked

✓ Generated the full monolith in one coordinated pass — 40+ procs, consistent schema, no drift
✓ Three-level CLAUDE.md kept Claude context-aware between legacy and new-service work
✓ Plan Mode before any schema or proc generation aligned on architecture before writing anything irreversible
✓ Five parallel agents for the backlog — under 10 minutes for 86 stories with real AC

What Surprised Us

⚡ Claude flagged coupling between billing and shipment proc groups unprompted — effectively a first-pass seam analysis
⚡ SYSTEM_NOTES.txt was eerily realistic — complete with passive-aggressive warnings about the batch scheduler
⚡ Static analysis quality of the as-is docs rivalled what a human architect would produce in a full week

Where It Saved Most Time

⏱ Boilerplate eliminated: ADO.NET forwarding, web.config wiring, SQL schema — hours → minutes
⏱ Static analysis: turned 42 procs + 5 triggers into structured spec in one pass
⏱ Backlog generation: what would be days of story-writing became a parallel 10-minute run
Technique Used

Three-level CLAUDE.md — user, project, and lagacy/ directory-specific constraints

Technique Used

Parallel subagents with explicit context passed per Task — one agent per two backlog features

Technique Used

Plan Mode for irreversible decisions · ADR-first discipline before any implementation work
